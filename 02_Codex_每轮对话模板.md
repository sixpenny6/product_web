# Codex 每轮对话模板

## 1. 首轮读取文档

```txt
请先读取 AGENTS.md 和 docs/tubular_motor_website_docs 中的全部 Markdown 文档。

本轮不要写代码，不要创建文件，不要运行命令。

请输出：
1. 项目一句话定位；
2. 目标用户；
3. MVP 功能范围；
4. 页面结构；
5. 前端模块；
6. 后端/CMS 模块；
7. 3D/360 查看器实现方案；
8. 推荐开发阶段；
9. 第一阶段应该做什么；
10. 风险点和需要我补充的资料。

严格要求：
- 不要编造产品参数；
- 不要一次性做完整项目；
- 不要跳到 CMS / 3D / 部署。
```

## 2. 阶段执行通用模板

```txt
现在执行阶段 X：[阶段名称]。

请严格遵守：
1. 只完成本阶段，不要做下一阶段；
2. 先说明你准备修改哪些文件；
3. 先说明你准备运行哪些命令；
4. 再进行修改；
5. 修改完成后运行必要检查；
6. 最后输出：
   - 完成内容
   - 修改文件
   - 如何运行
   - 如何测试
   - 已知问题
   - 下一步建议

项目规则：
- 不允许编造管状电机真实参数；
- 未知参数写 TODO 或 待补充；
- CTA 按钮必须清楚；
- 3D 模块必须有 fallback；
- 移动端必须可用；
- 不做在线支付。
```

## 3. 阶段结束 Review

```txt
请 review 当前阶段的改动。

重点检查：
1. 是否违反 AGENTS.md；
2. 是否违反 docs/tubular_motor_website_docs；
3. 是否编造产品参数；
4. 是否超出当前阶段；
5. TypeScript 类型是否合理；
6. 是否有 loading/error/empty 状态；
7. 移动端是否可用；
8. CTA 是否清楚；
9. 3D 模块是否有 fallback；
10. 是否可以 commit。

不要重写代码，先列问题和建议。
```

## 4. 报错修复

```txt
运行 [命令] 后出现以下错误。请只修复这个错误，不要重构无关代码，不要实现新功能。

错误如下：

[粘贴完整错误]
```

## 5. 防止 Codex 跑偏

```txt
停止扩展。你现在超出了当前阶段范围。

请回到阶段 X 的目标，只保留与本阶段相关的改动。
不要实现 CMS / 3D / 表单 / 部署，除非当前阶段明确要求。
请列出哪些改动应该撤回。
```
