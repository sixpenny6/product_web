# 360°/3D 产品查看器技术方案

## 1. 目标

为管状电机产品提供可交互的 360°查看体验，让客户能清楚理解：

- 电机整体外观
- 电机头部结构
- 线缆位置
- 安装支架
- 管径适配
- 限位调节位置
- 伺服反馈接口
- 控制接口
- 尺寸比例

## 2. 推荐实现路线

### 路线 A：真实 3D 模型，推荐

使用 GLB/gltf 模型，通过 React Three Fiber 渲染。

优点：

- 可自由旋转、缩放；
- 可添加热点；
- 可做 exploded view；
- 可未来扩展 AR；
- 更高级，接近 Dell PowerEdge XR5610 的体验。

缺点：

- 需要建模和模型优化；
- 对性能要求更高。

### 路线 B：360 图片序列，作为降级方案

用 36/72 张产品渲染图或照片实现拖动旋转。

优点：

- 稳定；
- 移动端兼容好；
- 开发更简单；
- 对模型要求低。

缺点：

- 不能自由查看细节；
- 热点和缩放体验较弱；
- 不够高级。

### 最佳方案

核心产品用路线 A，其他产品先用路线 B。

建议优先做 4 个核心模型：

1. Small Tubular Motor
2. 59 Series Motor
3. 92 Series Motor
4. 92 Servo Motor

## 3. 3D 资产制作流程

### 3.1 输入资料

需要从工厂或工程团队拿到：

- CAD / STEP / IGES 文件
- 产品尺寸图
- 产品照片
- 材质参考
- Logo 和铭牌位置
- 线缆/接口/支架细节
- 每个型号的差异点

### 3.2 模型处理

流程：

1. CAD 模型清理
2. 删除内部不可见结构
3. 控制面数
4. 拆分可点击部件
5. 设置材质
6. 导出 GLB
7. 压缩模型
8. 生成低模 LOD
9. 生成 poster 图片
10. 可选导出 USDZ 用于 AR

### 3.3 模型命名规范

```
/assets/models/
  small-tubular-motor-v1.glb
  series-59-motor-v1.glb
  series-92-motor-v1.glb
  series-92-servo-motor-v1.glb
```

### 3.4 模型性能目标

| 项目 | 目标 |
|---|---|
| 单个 GLB 文件 | 最好小于 5MB，复杂模型不超过 12MB |
| 首屏加载 | 先显示 poster，模型懒加载 |
| 移动端帧率 | 尽量 30fps 以上 |
| 桌面端帧率 | 尽量 60fps |
| 贴图 | 使用压缩纹理 |
| 多模型页面 | 不同时渲染多个 Canvas |

## 4. 查看器功能清单

### MVP 功能

- 加载 GLB 模型
- OrbitControls 拖拽旋转
- 限制过度缩放
- 自动旋转开关
- Reset View
- Fullscreen
- Loading 状态
- Poster fallback
- Hotspot 点击
- 右侧信息面板
- 移动端触控支持
- WebGL 不可用时显示图片序列或静态图

### 第二阶段

- Exploded View
- X-Ray View
- Dimensions Overlay
- AR 查看
- 颜色/配件切换
- 截图分享
- 与产品变体联动
- 3D 注释动画
- 旋转到指定热点视角

## 5. 热点数据结构

```ts
type ProductHotspot = {
  id: string
  label: string
  description: string
  position: [number, number, number]
  targetCamera?: {
    position: [number, number, number]
    lookAt: [number, number, number]
  }
  linkedSpecKey?: string
  imageUrl?: string
}
```

示例：

```json
{
  "id": "limit-adjustment",
  "label": "Limit Adjustment",
  "description": "Allows accurate upper and lower travel limit setup during installation.",
  "position": [0.22, 0.12, 1.4],

}
```

## 8. 用户体验细节

### 8.1 首次引导

第一次进入产品详情页：

- 显示一个小提示：“Drag to rotate”
- 3 秒后淡出
- 用户点击热点后不再显示

### 8.2 低性能处理

如果设备性能弱：

- 关闭阴影
- 降低 DPR
- 关闭环境反射
- 禁止自动旋转
- 改用图片序列

### 8.3 移动端处理

- Canvas 高度 420-560px
- 全屏按钮明显
- 热点面板从底部弹出
- 不要让 3D 区域阻止页面滚动太久

## 9. 和 CMS 的关系

后台产品编辑页需要配置：

- GLB 模型 URL
- USDZ 模型 URL，可选
- Poster 图
- 360 图片序列，可选
- 热点数组
- 模型默认缩放
- 模型默认相机位置
- 可用查看模式

## 10. 验收标准

- 用户可以在桌面端拖拽旋转产品。
- 用户可以在移动端滑动旋转产品。
- 热点可点击并显示说明。
- 3D 模型加载失败时有 fallback。
- 产品页面不因为 3D 模型卡死。
- 3D 模块不会挡住 CTA。
- 后台可以给不同产品配置不同模型和热点。
